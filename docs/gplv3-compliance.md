# GPLv3/ERPNext 라이선스·상표 준수 가이드라인

`CLAUDE.md` 절대 원칙 #4("GPLv3/상표 준수")의 상세 버전. 요약만 알면 되는 경우 CLAUDE.md의
한 줄 요약으로 충분하고, 실제로 의존성을 추가하거나 배포 계획이 생겼을 때만 이 문서를 연다.

**라이선스는 컴포넌트마다 다르다 — "Frappe/ERPNext는 GPLv3"로 뭉뚱그리면 틀린다:**

| 컴포넌트 | 라이선스 |
|---|---|
| Frappe Framework | **MIT** |
| ERPNext (코어 앱) | **GPL-3.0** |
| Frappe HR (`hrms`) | **GPL-3.0** |
| frappe_docker | MIT |
| Frappe 에코시스템 앱(Insights·Builder·CRM·Drive 등, 이 프로젝트엔 미설치) | AGPL-3.0 |

이 프로젝트가 쓰는(또는 쓸 예정인) 조합(Frappe+ERPNext+hrms+`babipa_erp` — 2026.07.20 기준
`hrms`는 아직 미설치, `docs/decisions.md` 참조)에는 AGPL 컴포넌트가 없다.
GPLv3든 AGPL이든 "네트워크로 서비스만 제공(사내 서버에서 돌리기만)"하는 것 자체는 소스 공개
의무를 발생시키지 않는다 — 의무는 **"conveying"(외부에 소스/바이너리를 넘기는 행위)** 시점에만
발생한다(AGPL은 네트워크 서비스 제공도 트리거하지만, 위 표대로 이 프로젝트엔 해당 컴포넌트가
없으므로 무관). 이 프로젝트는 사내 인트라넷 전용 self-host이므로 지금 당장은 대부분 조항이
적용되지 않지만, 지키지 않으면 **제3자 배포 시점(예: AU Co. 납품)에 되돌리기 어려운 문제**가
된다. 그래서 지금 단계에서부터 습관화해야 하는 규칙과, 배포 시점에만 발동하는 규칙을 구분해 둔다.

> 근거: `docs/research/ERPNext_가이드_검토_및_v16_빌드권고_2026-07-16.md` §3-⑧ (2026.07.20
> 정정 반영 — 이전 버전은 "Frappe/ERPNext는 GPLv3(일부 AGPL 요소 검토 필요)"로 부정확하게
> 서술돼 있었다).

## 지금부터 지켜야 하는 것 (사내 전용 단계에도 적용)

1. **코어 미수정** — `CLAUDE.md` 절대 원칙 #1과 동일. `bench update` 충돌 방지뿐 아니라,
   코어(GPL 소스)와 커스텀 앱(우리 소스)의 경계를 명확히 유지해 향후 배포 시 "무엇이
   우리 저작물이고 무엇이 GPL 소스인지"를 쉽게 구분할 수 있게 한다.
2. **상표 사용 금지** — "ERPNext"·"Frappe" 명칭/로고는 Frappe Technologies Pvt. Ltd.의
   상표다. 제품명·로그인 화면·문서·마케팅 자료 어디에도 공식 제휴/인증을 암시하는 방식으로
   쓰지 않는다. `babipa_erp` 같은 자체 브랜드명은 문제없지만 "ERPNext babipa Edition"류
   표기, 공식 로고를 재색상만 바꿔 재사용하는 것은 금지. `frappe-branding` 에이전트가 담당.
3. **저작권/라이선스 고지 보존** — 코어 파일을 커스텀 앱에 복사해 붙여넣는 것은 원칙적으로
   지양(원칙 #1 위반)하지만, 참고용으로 일부를 인용해야 한다면 원본의 GPL 고지·저작권
   헤더를 삭제하지 않는다.
4. **제3자 의존성 라이선스 확인** — 커스텀 앱에 새 Python(pip)/JS(npm) 패키지를 추가할 때
   GPLv3와 충돌하는 라이선스(재배포·수정 금지형 proprietary 등)가 아닌지 확인한다.
   `frappe-reviewer` 에이전트의 리뷰 항목에 포함되며, `CLAUDE.md` 체크리스트에도 반영.

## 제3자 배포 시점부터 지켜야 하는 것 (예: AU Co. 등에 동일 빌드 납품)

배포(conveying)가 발생하는 순간부터 GPLv3 조건이 실제로 발동한다:

5. **전체 소스 제공 의무** — 커스텀 앱뿐 아니라 결합 저작물(combined work) 전체, 즉
   Frappe+ERPNext 코어를 포함한 전체 소스를 제공하거나, 3년 이상 유효한 서면 제공
   확약을 해야 한다.
6. **라이선스 전문·저작권 고지 보존** — 배포물에 GPLv3 전문과 각 파일의 저작권 고지를
   유지해야 한다.
7. **변경 사항 고지** — 원본(ERPNext) 대비 무엇을 바꿨는지 명시하는 고지가 필요하다
   (GPLv3 §5). 이 프로젝트에서는 `docs/decisions.md`와 `docs/build-log.md`가 이 역할을
   상당 부분 겸할 수 있다.
8. **동일 권리 보장** — 수령자에게도 동일한 GPLv3 권리(추가 수정·재배포 등)를 보장해야
   하며, 추가 제약(재배포 금지 등)을 덧붙일 수 없다.

**배포 계획이 조금이라도 생기면 실제 조치 전에 반드시 사용자에게 먼저 확인한다** —
`CLAUDE.md`도, 이 문서도 그 조치를 사전 승인하지 않는다.

## 범위 밖

Frappe Cloud/Marketplace 게시 관련 정책(앱 심사, 유료 결제 연동 등)은 자체 호스팅에는
적용되지 않으므로 이 프로젝트에서 다루지 않는다.
